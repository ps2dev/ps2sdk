1 # PS2SDK - KERNEL LIBRARY
2 
3 ## SUMMARY
4 
5 This folder creates the `libkernel.a` library, which contains all the references to the BIOS functionality embedded in the hardware of every single PS2.
6 
7 Probably you've heard, that the `PS2 BIOS` is full of bugs, which totally accurate, however, these bugs aren't blocker you still could execute some program (with limited functionality) without any problem.
8 
9 ## SOLVING BIOS ISSUES
10 
11 In order to fix all these issues, we're patching the problematic functions by putting some wrappers in the middle before calling the actual BIOS function.
12 
13 Some of these wrappers are set during the initialization process of the application `_InitSys` function.
14 Additionally, we also use `_InitSys` to initialize some other functionality that the user may need.
15 
16 
17 ## CONSEQUENCES
18 
19 All these functions are set during the initialization process of every single binary, this implies unfortunately having a bigger size binary.
20 
21 Some apps require to have really tiny sizes, and also you may want to don't patch these BIOS functions, therefore we are offering the possibility to disable the functions called during the `_InitSys` function.
22 
23 Let's first take a look at the current `InitSys` method:
24 
25 ```c
26 void _InitSys(void)
27 {
28  StartTimerSystemTime();
29  InitTimerAlarm();
30  InitAlarm();
31  InitThread();
32  InitExecPS2();
33  InitTLBFunctions();
34 }
35 ```
36 
37 The nature of the functions called can be split into 2 different sections:
38 
39 1. Patched functionality
40 ```c
41  InitAlarm();
42  InitThread();
43  InitExecPS2();
44  InitTLBFunctions();
45 ```
46 
47 2. Additional timer utils
48 ```
49  StartTimerSystemTime();
50  InitTimerAlarm();
51 ```
52 
53 ## NO PATCHED BINARIES AND/OR TINY BINARIES
54 
55 On the `kernel.h` file at the very end, we have a section where we defined some `macros` for making it super easy the disable these functions.
56 
57 Let's put an example of how to disable the patched functionality:
58 
59 ```c
60 .....
61 #include <kernel.h>
62 #include <stdio.h>
63 
64 // With this line we disabled the patched functionalities
65 DISABLE_PATCHED_FUNCTIONS();
66 
67 int main(int argc, char *argv[])
68 {
69  printf("Hello world!\n");
70  return 0;
71 }
72 ```
73 
74 Let's put another example where we disable the extra timer functionality:
75 
76 ```c
77 .....
78 #include <kernel.h>
79 #include <stdio.h>
80 
81 // With this line we disabled the extra functionalities for timers
82 DISABLE_EXTRA_TIMERS_FUNCTIONS();
83 
84 int main(int argc, char *argv[])
85 {
86  printf("Hello world!\n");
87  return 0;
88 }
89 ```
90 
91 Therefore we can combine both:
92 
93 
94 ```c
95 .....
96 #include <kernel.h>
97 #include <stdio.h>
98 
99 // With this line we disabled the patched functionalities
100 DISABLE_PATCHED_FUNCTIONS();
101 // With this line we disabled the extra functionalities for timers
102 DISABLE_EXTRA_TIMERS_FUNCTIONS();
103 
104 int main(int argc, char *argv[])
105 {
106  printf("Hello world!\n");
107  return 0;
108 }
109 ```
110 
111 ## HOW IS THIS WORKING?
112 
113 It honestly is quite easy.
114 All these functions that we may want to disable are defined as `__attribute__((weak))`, so it means that if a "strong" implementation is defined, they will override.
115 For this reason, the macros defined in the `kernel.h` are helpers that either does dummy implementations or bypass the patched wrappers.
